# The Process Global Object

It lets you access properties of the current running process.

```javascript
process.pid // gives the pid assigned by OS to the process.

process.title // gives title
process.title = 'new title' // setting a new title

process.version // node version 'v22.01.21'
process.versions // → { node: '22.1.0', v8: '...', uv: '...', ... }
// v8 is the JS engine, uv is libuv — the C library powering Node's event loop

process.platform // 'linux', 'darwin' (macos), 'win32'

process.memoryUsage()
// → { rss: 30mb, heapTotal: 6mb, heapUsed: 4mb, external: 1mb }
// rss = total memory the OS gave your process
// heapUsed = what your JS objects are actually consuming
// Useful for debugging memory leaks

process.uptime()   // → 143.2  // seconds since the process started

process.cwd() // gives current working dir root

process.exit(0) // graceful end / exit process
process.exit(1) // exit with error, signals to the OS something went wrong
```

## Environment Variables

```javascript
process.env.API_KEY // reads env injected by OS / container
```

To read `.env` file:
```javascript
import 'dotenv/config' // automatically injects .env variables into process.env
```

## Event Handlers for process

```javascript
process.on('event', callback_fn)

process.on('exit', (code) => {
  // Runs synchronously just before process dies
  // Can't do async here — it's too late
  console.log(`Exiting with code ${code}`)
})

process.on('uncaughtException', (err) => {
  // Synchronous throw that was never caught
  console.error(err)
  process.exit(1)
})

process.on('unhandledRejection', (err) => {
  // Async/await throw that was never caught
  console.error(err)
  process.exit(1)
})

process.on('SIGTERM', () => {
  // OS sent a termination signal (e.g. deployment, docker stop)
  // Graceful shutdown — close DB connections, finish in-flight requests
  server.close(() => process.exit(0))
})
```
