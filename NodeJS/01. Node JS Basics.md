# What is Node JS?

Its the same browser JS engine, but built for systems to interact with OS and files.

The browser's JS runs in a sandbox, it can't touch your files, OS, or open network sockets.

Node JS is the same JS language under the hood but it has access to your (fs, os, path, http).

## The Module System in Node JS

We use the latest ES Modules (ESM) Import/Export in Node and Express. Tho you might see old CJS (Common JS) module syntax too.

### ES Modules Syntax
```javascript
import module_name from 'filepath'
import { module_name } from 'filepath'

export const some = ...
export default ...
```

### Older CJS syntax
```javascript
const module_name = require('module_name') // import

module.exports = { module_name, ... } // export
```

eg:
```javascript
const fs = require('fs')
```

> **Side note:** For older node projects (<Node v22.7)
> You can use ESM in Node by either naming your file `.mjs` or setting `"type": "module"` in `package.json`.
>
> For Express projects, most people stay on CommonJS or use ESM via the `package.json` flag. Just know both exist and don't mix them.
