# Filesystem (fs) module

```javascript
import { readFile, writeFile, appendFile, unlink, mkdir, readdir, access } from 'node:fs/promises'
```
> Don't import from only `fs`, it gives the old callback based api.

## Reading

```javascript
const data = await readFile('config.json', 'utf8')
```
First arg is filepath, second is encoding. Without encoding you get a raw `Buffer`, not a string.

If the file doesn't exist, it throws. Always wrap in try/catch or let it bubble up to your error handler.

```javascript
try {
  const data = await readFile('config.json', 'utf8')
  const config = JSON.parse(data)
} catch (err) {
  console.error('config missing:', err.message)
}
```

## Writing
```javascript
await writeFile('output.txt', 'hello') 
```
Creates a new file if doesn't exist, overwrites anything in the file before.

## Appending
```javascript
await appendFile('logs.txt', `${new Date().toISOString()} - server started\n`)
```
Appends, doesn't overwrite.

## Deleting
```javascript
await unlink('temp.txt')
```

## Directories

### `mkdir`
```javascript
await mkdir('logs', { recursive: true })
// recursive: true = won't throw if folder already exists
// Without it — throws if folder exists
```

### `readdir`
Provides an array of all the sub dir and files in the dir.
```javascript
const files = await readdir('./public')
// → ['index.html', 'style.css', 'app.js']
// Just filenames, not full paths
```

## Checking if something exists

```javascript
try {
  await access('file.txt')
  // exists
} catch {
  // doesn't exist
}
```

## Why async matters?
If we read a file on a server, it would block the thread of JS, and since JS is single threaded, everything would freeze until file operation finishes.

There is an explicitly synchronous read/write method too, don't use it until needed.
```javascript
import { readFileSync } from 'node:fs'
const config = JSON.parse(readFileSync('config.json', 'utf8'))
```

> **The old CJS fs callback way:** (just for reference)
> ```javascript
> const fs = require('fs')
> fs.readFile('file.txt', (err, data) => { ... })
> ```
