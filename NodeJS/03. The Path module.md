# Path Module

Node JS path module basically helps to manage diff path system for different OSes. File paths are different on Windows (`\`) vs Mac/Linux (`/`). If you hardcode `/` you'll break someone's Windows machine.

```javascript
// import
import path from 'node:path' 
```
> **Gotcha:** Prefixing with `node:` is the modern best practice to avoid conflicts with npm packages of the same name.

## Methods

### `path.join()`
Joins given strings or semi-paths into a single path.
```javascript
path.join('Project', 'react', 'Caden', 'file.txt')
// => 'Project/react/Caden/file.txt' (on macos and linux)
// => 'Project\react\Caden\file.txt' (on windows)
```

### `path.resolve()`
Joins and provides an absolute path from the cwd to the given path.
```javascript
path.resolve('folder', 'sub', 'file.txt')
// => 'C:\Users\Username\Projects\Backend\folder\sub\file.txt' (in windows)
// => '/home/user2/Projects/Backend/folder/sub/file.txt' (in mac and linux)
```
> `'C:\Users\Username\Projects\Backend'` is the current working directory (cwd).

### `path.basename(path)`
Gives the file name at the end of the path.
```javascript
path.basename('users/user2/app/index.js') // => 'index.js'
```

### `path.dirname(path)`
Gives the dir of the file path.
```javascript
path.dirname('users/user2/app/index.js') // => 'users/user2/app'
```

### `path.extname(path)`
Gives the ext of the file at the end of filepath.
```javascript
path.extname('users/user2/app/index.js') // => '.js'
```

### `path.parse(path)`
Breaks the path into object keys.
```javascript
path.parse('/users/user2/app/index.js')
// => { root: '/', dir: '/users/user2/app', base: 'index.js', ext: '.js', name: 'index' }
```

## Getting the current working filename & directory

We use `import.meta.url` (which provides current file name as url `'file:///home/val/app/index.js'`). Then we use the `fileURLToPath` method to convert it into `'/home/val/app/index.js'`.

```javascript
import { fileURLToPath } from 'node:url'
import path from 'node:path'

const filename = fileURLToPath(import.meta.url)
```

To get the current working directory:
```javascript
const dirname = path.dirname(filename)
```
or use `process.cwd()` to get the root dir only.

> **Gotcha:** `process.cwd()` is where the script was *run from*, while `path.dirname(filename)` is where the file *lives*.
