# 04.1. The Response Object

When defining routes, the callback gets a `response` (res) object:

```js
app.get(route, (request, response) => { ... })
```

## `res.send()`

Sends data back. Accepts a string, object, or array — no confusion here.

```js
res.send(data) // string | object | array
```

> By default, if you don't set a status, `res.send()` defaults to `200 OK`.

## `res.status()` vs `res.sendStatus()`

### `res.status(code)`

Sets the HTTP status code — but **doesn't send the response**. The request will keep loading if you stop here.

Use it by chaining with `.send()`:

```js
res.status(200).send(data)
res.status(400).send({ error: 'Bad Request. Fuck off' })
```

### `res.sendStatus(code)`

Sets the status **and** sends the response as the corresponding status message string. No chaining, no custom message.

```js
res.sendStatus(400) // → "Bad Request"
res.sendStatus(404) // → "Not Found"
```

> `sendStatus` = `status() + send()` shorthand, but you can't customize the message body.
