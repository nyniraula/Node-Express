# 06. POST Requests

A POST request is an HTTP request where the client sends data **to** the server — like submitting a form.

The data you send is called a **request body** or **payload**.

---

## Creating a POST Route

Same as `app.get()`, just `app.post()`:

```js
app.post('/users', (request, response) => {
  const payload = request.body;
});
```

A POST route can share the same path as a GET route — they're separate because the HTTP method is different. So `/users` GET and `/users` POST can both exist.

---

## Parsing the Body

`request.body` won't work out of the box. You need this middleware at the top (right after `const app = express()`):

```js
app.use(express.json());
```

This tells Express: if a request comes in with `Content-Type: application/json`, auto-parse the body into a JS object.

> Middleware is covered later — for now just know this line needs to be there.

---

## What a POST Route Usually Does

1. Get the incoming data from `request.body`
2. Add it to a DB (or forward it somewhere)
3. Respond with status `201 Created` — and optionally return the new data

```js
app.post('/users', (request, response) => {
  const userData = request.body;

  const newUser = { id: users.length + 1, ...userData };
  users.push(newUser);

  response.status(201).send(users);
});
```
