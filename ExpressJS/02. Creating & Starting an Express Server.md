# 02. Creating & Starting an Express Server

## The Basics

```js
import express from 'express'

const app = express()   // initializes the server

app.listen(3000)        // starts it, listens on port 3000
```

## PORT from Environment

Don't hardcode the port — read it from `process.env.PORT` and fall back to a default:

```js
const PORT = process.env.PORT || 3000
```

> Hardcoding ports leaks info about your server setup. Not ideal.

## Full Minimal Setup

```js
import express from 'express'

const PORT = process.env.PORT || 3000

const app = express()

app.listen(PORT, () => {
  console.log(`${new Date().toISOString()}: Server Started, Running on PORT ${PORT}`)
})
```

## Startup Callback

`app.listen()` takes an optional second argument — a callback that runs once the server is up. Good spot for a startup log.

```js
app.listen(PORT, () => {
  console.log(`${new Date().toISOString()}: Server Started, Running on PORT ${PORT}`)
})
