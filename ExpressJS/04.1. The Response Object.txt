When defining routes, you seen a callback with response object:

app.get(route, (request, response)=>{...})

There are multiple properties and methods for it, Ill explain the main confusing ones.

response.send(data:str|obj|arr)

just sends the given data, no any confusion.


response.status(code) vs response.sendStatus(code).

response.status(400), just sets the response HTTP status to  the provided HTTP status code, but doesnt send the response, you have to do that manually.

It sets the HTTP status code to given code, but doesnt send response alone. The thing will keep loading.

So this is mainly used with chaining messages. and sending custom errors

response.status(200).send(data)

response.status(400).send({error:"Bad Request. Fuck off"})





response.sendStatus(code):

It sets the HTTP response status to provided one, and also sends the response as the corresponding message for that code.

Its like chaining response.status().response.send(), but you cant modify the error msg.

response.sendStatus(400) => this sets the response to 400 HTTP status code and also sends a string response of corresponding thing aka "Bad Request".

if it was 404 => It would say Not Found.





Side Note: Also by default, when using response.send() , if you dont specifiy a status, it will default to 200.
