# 07. PUT Requests

PUT is used when you want to **entirely replace** a record — not just update a field, the whole thing.

---

## Defining a PUT Route

```js
app.put('/user/:id', (request, response) => {
  const body = request.body;
  const parsedId = parseInt(request.params.id);

  if (isNaN(parsedId)) return response.sendStatus(400);

  const findUserIdx = users.findIndex((user) => user.id === parsedId);

  if (findUserIdx < 0) return response.sendStatus(404);

  users[findUserIdx] = { id: parsedId, ...body }; // completely replacing the previous record

  return response.send(users);
});
```

The key line is `{ id: parsedId, ...body }` — you're not merging, you're replacing. Old fields are gone unless `body` includes them.

> Don't use PUT for partial updates (e.g. just updating one field). That's what PATCH is for.
