by default, when we initialize and start the express server, it does nothing, when we go to localhost:PORT.

This is cause we havent defined routes and what happens when we go there.

we define a router by the request type it does ( GET, POST, PUT, PATCH, DELETE)

for now lets see a route that provides a data (get route)

we define it by .get() method on app

app.get(route, RequestHandler)

route is just a string for the route: '/' for the base route. '/...' for other


RequestHandler is basically a function, a callback function that is automatically called when we get request that route.

(request, response) => {
    return ....
}

the callback fn has two params request and response. request has properties for its header (meta data: origin, ...) and body if it sends any data.

The response object has methods and properties to define what to do after getting the request.

response.send(), you can send data like an api
response.send('Hello world') sends a string

you could even do json object or array.

response.send( {"name":"ny", "id":205224} )


you can send status codes too, file too.

response.status(300), just sends the corresponding status msg as response.


Laslty, we can even chain response method.

response.status(201).send({"name":"ny", "id":205224}) 

This sends a response of the json obj with a status code 201 (which can be seen as header in network tab dev tools)

so here is how it looks at total.

or the whole server:
import express from "express";

const app = express();

const PORT = process.env.PORT || 3000;

app.get("/", (req, res) => {
 return  res.send({
    name: "hello",
    idk: "what",
  });
});

app.listen(PORT, () => {
  console.log(
    `${new Date().toISOString()}: Server Started, Running on PORT ${PORT}`,
  );
});
