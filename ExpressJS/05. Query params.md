# 05. Query Params

You've probably seen URLs like this:

```
somelink.com/route?key=value
```

That `?key=value` is a query parameter. `?` signals that what follows is a query param. Keys and values are always strings.

Chain multiple params with `&`:

```
youtube.com/sidemen/videos?sort=popular&order=ascending
```

Here `sort = popular` and `order = ascending` are both query params.

Query params are just key-value pairs that give the server extra info about what you want — like filters, sort order, pagination, etc.

---

## Accessing Query Params (Server Side)

The request object has a `request.query` property — it's an object of key-value pairs.

```js
// ?sort=ascending
request.query // { sort: 'ascending' }

// ?sort=ascending&filter=popular
request.query // { sort: 'ascending', filter: 'popular' }
```

---

## Handling Query Params — 3 Cases

1. **Params not there (undefined)** → just send the data as-is
2. **Params there but wrong** → throw an error or send data as-is
3. **Params there and correct** → modify and send the data

```js
app.get('/users', (request, response) => {
  const { filter: qfilter, value } = request.query;

  // check if query params aren't there or filter key doesn't exist on users
  if (!(qfilter && value && qfilter in users)) {
    return response.send(users);
  }

  // filter and value are present and valid
  if (qfilter && value) {
    return response.send(
      users.filter((user) => {
        return user[qfilter].toLowerCase().startsWith(value.toLowerCase());
      }),
    );
  }
});
```

This is a `/users` endpoint with optional `filter` and `value` query params. When both are provided and valid, it filters users where the given property starts with the given value (case-insensitive). If either is missing or invalid, it just returns all users.

> All query param values come in as **strings** — parse them if you need numbers or booleans.
