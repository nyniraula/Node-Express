# 08. PATCH Requests

PATCH is used when you want to update **specific fields** of a record — not replace the whole thing.

---

## Defining a PATCH Route

```js
app.patch('/user/:id', async (request, response) => {
  const body = request.body;
  const parsedId = parseInt(request.params.id);

  if (isNaN(parsedId)) return response.sendStatus(400);

  const users = await readDB();

  const userIdx = users.findIndex((user) => user.id === parsedId);

  if (userIdx === -1) return response.sendStatus(404);

  users[userIdx] = { ...users[userIdx], ...body };

  await writeDB(users);

  return response.send(users[userIdx]);
});
```

`{ ...users[userIdx], ...body }` — spread the existing record first, then spread the new data on top. Any matching keys get overwritten, everything else stays.

> Only the fields you send in `body` get updated. The rest of the record is untouched.
