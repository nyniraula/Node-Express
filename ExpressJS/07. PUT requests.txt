PUT requests:

PUT HTTP method is basically used when we want to 'entirely' replace / update a record.

Suppose you have a users obj:

You want to replace the user with id 10 to a entierely new record (new user data).

This is when you use PUT.

Route is defined by:

app.put('/user/:id', (request, response) =>{
    const body = request.body

    const parsedId = parseInt(request.params.id)

     if (isNaN(parsedId)) {
    return response.sendStatus(400);
  }

  const findUserIdx = users.findIndex((user) => user.id === parsedId);

  
  if (findUserIdx < 0) {
    return response.sendStatus(404);
  }

  users[findUserIdx] = {id: parsedId, ...body} //we are completely replacing the previous record.

  return response.send(users);
} )


Dont use PUT request when you only want to update suppose a select field of a record. (partial update)


USE PATCH request for that purpose.

PUT request is only used, when you are sending a entire new record
