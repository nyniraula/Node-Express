# 03. Routes & GET Requests

By default the server does nothing at `localhost:PORT` — you have to define routes.

Routes map to HTTP methods: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`.

## Defining a GET Route

```js
app.get(route, (req, res) => {
  return res.send(...)
})
```

- `route` — a string like `'/'` or `'/users'`
- `req` — has headers, body (if sent), params, etc.
- `res` — has methods to send back a response

## `res.send()`

```js
res.send('Hello world')           // string
res.send({ name: 'ny', id: 205224 })  // JSON object or array
```

## Status Codes

```js
res.status(400)                            // just sets the status
res.status(201).send({ name: 'ny' })       // chain with send
res.sendStatus(404)                        // sends the code + default status message
```

> Check response status codes in the **Network tab → Headers** in dev tools.

## Full Example

```js
import express from 'express'

const app = express()
const PORT = process.env.PORT || 3000

app.get('/', (req, res) => {
  return res.send({ name: 'hello', idk: 'what' })
})

```
