# 04. Route Params & Dynamic Routes

Instead of returning all users, sometimes you want a specific one — by ID, name, etc. That's what route params are for. Same concept as dynamic routes in React Router.

## Syntax

Prefix the segment with `:` to make it a param:

```js
app.get('/users/:id', (req, res) => { ... })
```

Access it via `req.params` — it's always a **string**:

```js
req.params  // { id: '3' }
```

## Full Example

```js
const users = [
  { id: 1, name: 'Alice Mercer', age: 28 },
  { id: 2, name: 'James Okafor', age: 34 },
  { id: 3, name: 'Priya Nair', age: 22 },
  { id: 4, name: 'Carlos Vega', age: 41 },
]

app.get('/users/:id', (req, res) => {
  const parsedId = parseInt(req.params.id) // params is an object, values are always strings

  if (isNaN(parsedId)) {
    return res.status(400).send({ msg: 'Bad request. Invalid ID' }) // 400 = bad request
  }

  const user = users.find((el) => el.id === parsedId) // find user matching the id

  if (!user) {
    return res.sendStatus(404) // sends the code + default status message (Not Found)
  }

  return res.send(user) // found it, send it back
})
```

> `isNaN()` — "is Not a Number". Catches cases where someone passes `/users/abc`.

## Multiple Params

```js
app.get('/users/:id/:username', (req, res) => {
  const { id, username } = req.params
})
```
