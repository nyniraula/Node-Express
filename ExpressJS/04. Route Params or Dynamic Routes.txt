Route Params / Dynamic Routes:

previosly, we had define a users get route, which would give all the users:
`
const users = [
  { id: 1, name: "Alice Mercer", age: 28 },
  { id: 2, name: "James Okafor", age: 34 },
  { id: 3, name: "Priya Nair", age: 22 },
  { id: 4, name: "Carlos Vega", age: 41 },
];

app.get("/users", (req, res) => {
  res.send(users);
});

`


We can define dynamic routes with route params:

so basically get user/1 meaning get user with specific id. or specific name. that stuff

this is achieved by defining the route with a colon before it ('/user/:id'). Similar to React router dynamic routes.


to access the id, we can use request.params inside a callback function and it will give an object with the id and any other route params if present. 
{
    id: ...,
}

the param is a string always

app.get('/user/:id', (request, response)=>{
    //lets get the id as int
    const parsedId = parseInt(request.params.id) //params is an object.

    //check if id is a not number and throw an error.

    if (isNaN(parsedId)) { //NaN means Not a Number
        return response.status(400).send({msg:"Bad request. Invalid ID"}) //returning a bad request repsonse for id other than integer, with 400 status code header
    }

    const findUser = users.find((el)=>{
        el.id === id
    })

    if (!findUser) {
        return response.sendStatus(404) //send not found error status msg (not just the code)
    }

    return response.send(findUser)
})

---

You could also have multiple route params inside a single route.

app.get('users/:id/:username/, ...)

which could be accessed by request.params.id , request.params.username
